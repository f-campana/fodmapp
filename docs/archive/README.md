# Documentation Archive

This archive stores historical planning and review artifacts that are no longer active operating
guidance.

Archived docs are the canonical historical record for superseded work. Active directories should
not keep archive mirrors or archived stubs.

When historical context is intended, link directly to the archive path instead of routing through
an active directory.

Active documentation lives in `/docs` and in component-level READMEs or runbooks.

## Archive Structure

- `transition/`: completed planning, sequencing, state, and handoff artifacts.
- `reviews/`: resolved or superseded review reports retained for traceability.
