## When to use

Use ScrollArea for dense lists, preparation notes, or compact dashboards that
must stay inside a fixed block.

Avoid ScrollArea when the page can grow naturally without forcing a nested
scroll region.

## Composition patterns

The base composition works in normal page flow. Use the surfaced pattern when
the scroll region belongs inside a card or inset panel and border spacing needs
visual review.

## Accessibility contract

- the viewport stays keyboard reachable through its tab stop
- root, viewport, scrollbar, thumb, and corner slot hooks stay stable in default composition
- both scrollbar orientations remain mounted so visual regression checks can inspect the full surface

## Slot hooks

- in default composition, `data-slot` hooks stay stable on the root, viewport, scrollbar, thumb, and corner
- ScrollArea does not expose an `asChild` exception for these internal hooks

## Validation

What is checked automatically:

- Storybook runs an accessibility scan scoped to the ScrollArea example area
- Storybook interaction checks verify the viewport tab stop plus the mounted scrollbar, thumb, and corner hooks
- UI tests cover slot stability, keyboard reachability, ref forwarding, thumb token classes, and a11y

What is checked manually:

- visual pass for clipped tracks or thumbs in light and dark themes
- review for border collisions in surfaced and nested-card layouts

## Responsive notes

Long cards may wrap, and dual-axis overflow should stay readable without
clipping the viewport or scrollbar chrome.
