## When to use

Use Tooltip for short help text, quick portion reminders, or clarifying notes
that support an existing control.

Avoid Tooltip for long instructions, interactive content, or blocking
confirmation flows.

## Accessibility contract

- the trigger remains keyboard reachable and keeps focus after Escape closes the tooltip
- open content stays linked to the trigger through `aria-describedby`
- content can mount into a local container for isolated interaction and a11y checks

## Slot hooks

- in default composition, `data-slot` hooks stay stable on the provider, root, trigger, and content
- when using `TooltipTrigger` with `asChild`, child attributes take precedence, including `data-slot`

## Validation

What is checked automatically:

- Storybook runs an accessibility scan scoped to the local tooltip audit area
- Storybook interaction checks verify keyboard focus open, local container mounting, the `asChild` slot exception, Escape close, and trigger focus retention
- UI tests cover default-composition slot stability, the `asChild` slot exception, local container mounting, semantic classes, ref forwarding, and a11y

What is checked manually:

- visual pass for compact trigger-to-panel spacing and non-touching borders
- narrow-width review to confirm the open layer stays visible without clipped copy

## Responsive notes

Tooltip content keeps a compact width and collision padding so long copy can
wrap without hiding the panel at narrow widths.
