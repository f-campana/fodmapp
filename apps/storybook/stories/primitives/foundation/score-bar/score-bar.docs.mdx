## When to use

Use ScoreBar when the product already has a defined score model and users only
need a quick status read inside a larger summary.

Avoid it when users need a fully explained chart, a history view, or a precise
standalone measurement with no surrounding context.

## Threshold meaning

`ScoreBar` clamps incoming values to the 0..1 range and maps fill color as
follows:

- below `0.5`: danger
- `0.5` through `0.7`: warning
- above `0.7`: success

## Accessibility contract

- the root exposes `progressbar` semantics with the normalized value
- a visible label is used for `aria-labelledby`; otherwise provide an `aria-label`
- the component does not render a visible numeric value on its own

## Validation

What is checked automatically:

- Storybook runs an accessibility scan scoped to the score-bar audit area
- interaction checks verify value clamping and threshold mapping
- UI tests cover label wiring, threshold changes, clamping, numeric labels, and a11y

What is checked manually:

- review for label and nearby value readability on narrow surfaces
- review for threshold contrast in both themes

## Responsive notes

Keep the visible score text near the bar on compact widths so users do not have
to infer the numeric value from color alone.
