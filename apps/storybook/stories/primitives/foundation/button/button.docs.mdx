## When to use

Use Button for one clear action in the current surface, with a single primary
action per area when hierarchy matters.

Use `asChild` only when the child element already has the right semantics, such
as a link for navigation.

## Accessibility contract

- the default element is a native `button`
- the default `type` is `button`, not `submit`
- disabled state is native `disabled`, not a custom loading pattern
- icons and loading affordances are regular children, not a built-in API

## Validation

What is checked automatically:

- UI tests cover the default slot marker, consumer `data-slot` overrides,
  variant and size data attributes, disabled behavior, and `asChild` rendering
- Storybook runs an accessibility scan scoped to the local button audit area

What is checked manually:

- visual pass for focus rings, disabled contrast, and size variants in light and
  dark themes
