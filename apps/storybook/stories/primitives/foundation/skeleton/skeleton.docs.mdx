## When to use

Use Skeleton when the final layout shape matters and a lightweight placeholder
helps users anticipate the loaded content.

Avoid it as the only loading communication when the surrounding region needs a
spoken status.

## Accessibility contract

- Skeleton is decorative and should remain `aria-hidden`
- the parent region should own `aria-busy` or loading copy when users need
  broader status feedback

## Validation

What is checked automatically:

- Storybook verifies decorative rendering and `aria-hidden` behavior

What is checked manually:

- visual pass for placeholder rhythm, shimmer restraint, and layout stability
