## When to use

Use Chip when people need to toggle, review, or dismiss an active token inside
the same toolbar or filter surface.

Use Badge instead when the token is read-only status.

## Accessibility contract

- the primary chip trigger exposes pressed state through button semantics
- removable chips add a second dedicated remove button with its own label
- selection and removal are separate contracts and should stay visually distinct

## Validation

What is checked automatically:

- Storybook verifies pressed, removable, and disabled states in one filter-card
  context
- interaction checks cover trigger/remove callbacks and stable slot markers

What is checked manually:

- visual pass for chip spacing, removable affordance clarity, and pressed-state
  contrast
