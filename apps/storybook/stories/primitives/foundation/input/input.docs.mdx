## When to use

Use Input for single-line text entry when the field should stay compact and the
surrounding form provides the label, helper text, and validation copy.

Use Textarea instead when the user needs visible multi-line entry.

## Accessibility contract

- the component is a native `input`
- `aria-invalid` affects semantics and styling only; validation logic stays
  outside the component
- the component exposes one stable `data-slot` hook for layout and styling
- labels, helper text, and error copy should come from surrounding markup or a
  higher-level field wrapper

## Validation

What is checked automatically:

- UI tests cover label association, stable slot hooks, invalid styling, refs,
  disabled state, and a11y
- Storybook runs an accessibility scan scoped to the local input audit area

What is checked manually:

- visual pass for field text, realistic placeholder hints, helper copy, and
  focus rings at compact widths in light and dark themes
