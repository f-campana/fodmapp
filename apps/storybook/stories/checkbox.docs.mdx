## When to use

Use Checkbox for consent, saved preferences, or optional follow-up actions that
users can enable independently.

Avoid Checkbox when choices are mutually exclusive or when users must pick
exactly one option from a set.

Pair the control with a real label and supporting text. When the choice belongs

## Accessibility contract

- the control exposes the native checkbox pattern with `aria-checked`
- the label remains discoverable through the linked `htmlFor` relationship
- invalid state is exposed through `aria-invalid`
- focus treatment stays visible on the active surface
- the hit target stays touch-friendly while the label can wrap naturally

## Validation

What is checked automatically:

- Storybook runs an accessibility scan scoped to the Checkbox example area
- Storybook interaction checks verify checked-state changes and slot presence
- UI tests cover click behavior, disabled behavior, stable slot hooks, ref forwarding, and a11y

What is checked manually:

- visual pass for label alignment, cursor behavior, and focus visibility
- mobile smoke pass for long-label wrapping without horizontal overflow

## Responsive notes

Long labels should wrap next to the control without collapsing the vertical
alignment or pushing the control off the first line.
