## When to use

Use RadioGroup when the available choices are few enough to compare directly and
the selected option should stay visible without opening another layer.

Avoid RadioGroup when the option list is long, searchable, or better grouped
behind a trigger.

## Accessibility contract

- the root is exposed as a real `radiogroup`
- each item exposes `radio` semantics with a single checked option
- labels stay discoverable through `htmlFor` and visible text
- keyboard users can move across the set while preserving single selection
- focus remains visible on each item without reducing legibility

## Validation

What is checked automatically:

- Storybook runs an accessibility scan scoped to the RadioGroup example area
- Storybook interaction checks verify semantic selection changes on the rendered surface
- UI tests cover controlled and uncontrolled state, keyboard tab-stop behavior, disabled items, stable slot hooks, refs, and a11y

What is checked manually:

- visual pass for label spacing, checked indicator contrast, and cursor behavior
- narrow-width pass to confirm long labels wrap without breaking alignment

## Responsive notes

Allow labels to wrap freely. The selected indicator should stay aligned to the
first line and the group should not introduce horizontal scrolling.
