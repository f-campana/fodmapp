# Fodmap Reporting Checklist

Every delivery should include:

1. Changed files and why.
2. Validation commands run and their outcomes.
3. Remaining risks, assumptions, or follow-up work.

## Suggested Output Template

```text
Changed Files
- path/to/file: reason

Validation
- command: pass/fail (+ key result)

Remaining Risk / Follow-up
- risk or follow-up item
```
